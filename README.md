# QueryQuest Weekly Assignments

Welcome to the **QueryQuest Weekly Assignments** repository. This repository is used for submitting and tracking weekly assignments.

## A Quick Note

The weekly assignments are non-evaluative and are meant to help you learn, experiment and stay consistent throughout the program.

Feel free to discuss ideas and approaches with your peers, but please make sure that the final submission you upload is your own work. If the same assignment is submitted by multiple participants under different names, all involved participants will not be given ratification, even if they complete the Mid-Term and End-Term evaluations.

## Submission Guidelines

### File Naming Convention

All assignment submissions must follow the naming format:

```text
rollnumber_FirstNameLastName.<extension>
```

**Example:**

```text
230627_ManasTodi.pdf
230627_ManasTodi.zip
```

The required file type (`.pdf` or `.zip`) will be specified in the assignment instructions. Please ensure you submit the correct file format for each assignment.

### Folder Structure

Each weekly assignment will have its own folder in the repository.

Example:

```text
QueryQuest/
├── Assignment_01/
├── Assignment_02/
├── Assignment_03/
└── ...
```

Upload your submission to the corresponding assignment folder.

Example:

```text
Assignment_01/
└── 230627_ManasTodi.pdf
```

## How to Submit Your Assignment

### Step 1: Fork the Repository

1. Open the repository on GitHub.
2. Click the **Fork** button in the top-right corner.
3. This creates your own copy of the repository.

### Step 2: Upload Your File

1. Open your forked repository.
2. Navigate to the appropriate assignment folder.
3. Click **Add file → Upload files**.
4. Upload your submission (`.pdf` or `.zip` as specified in the assignment).
5. Click **Commit changes**.

### Step 3: Create a Pull Request

1. In your fork, click **Contribute** → **Open Pull Request**.
2. Verify that your submission has been added to the correct assignment folder.
3. Click **Create Pull Request**.

## Important Notes

* Follow the naming convention exactly.
* Submit only to the designated assignment folder.
* Ensure your file opens correctly before uploading.
* One submission per assignment unless instructed otherwise.
